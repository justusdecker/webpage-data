# 🐱‍👤 Python-QuickSQL (QSQL): Modellierung ohne Boilerplate-Code!
![logo](https://media.licdn.com/dms/image/v2/D4D22AQHZDR_Cxh10yQ/feedshare-shrink_800/B4DZv4BSD.G4Ag-/0/1769392644502?e=1774483200&v=beta&t=0c_otXf-W4OAEZEB23i_omWafDdrNcPZKKmYm-9Ulj8)

Ich freue mich, euch mein Projekt QuickSQL (QSQL) vorzustellen.

Falls ihr euch jemals von sich wiederholenden SQLAlchemy-Setups und endlosen Python-Attributdefinitionen erdrückt gefühlt habt, ist QSQL genau das Richtige für euch.

Es handelt sich um eine Template-Skriptsprache, die eine übersichtliche, tagbasierte Syntax in Python-SQLAlchemy-kompatiblen Code umwandelt.

Warum QuickSQL?
* Intuitive Syntax: Definiert Tabellen, Datenbanken und Skripte mithilfe eines einfachen, lesbaren Tag-Systems.
* Lebenszyklus-Hooks: Ordnet Aktionen wie on_commit oder on_create direkt dem Schema zu.
* Python-Integration: Bindet native Python-Logik mit `<Custom>`-Blöcken ein und verwaltet Erweiterungen über `<Use>`.
* Sofortiges Mocking: Integrierte Befehle wie `<FillRandom>` und `<Output>` für schnelles Testen und Prototyping.

Funktionen:
* Benutzerdefinierte Syntaxhervorhebung für VS Code
* Leistungsstarker .qsql-zu-.py-Konverter
* Automatisierte Befehle zum Erstellen und Befüllen der Datenbank
